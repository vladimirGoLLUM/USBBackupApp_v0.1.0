# USB Backup App — пакет для передачи клиенту

## Что входит в продажный архив

Скрипт в **корне репозитория** [`build_sale_package.py`](../build_sale_package.py) собирает ZIP со структурой:

- `USBBackupApp.exe` — из `source/dist/USBBackupApp.exe`
- `README_КЛИЕНТУ.md` (собирается из `README_RU.md`), `README_CLIENT.md` — инструкции (RU / EN)
- `LICENSE.txt` — MIT (копия корневого `LICENSE`)
- `CHANGELOG.md` — копия из `docs/CHANGELOG.md`
- `SUPPORT.md` — контакты поддержки
- `screenshots/` — скриншоты для маркетплейса
- `config/app_state.json.example` — шаблон поля «папка бэкапа» (без личных данных)

Имя архива: `USBBackupApp_Sale_vX.Y.zip` (версия задаётся параметром `--version`).

Дополнительно для разработчика: шаблон установщика [`USBBackupApp.iss`](../USBBackupApp.iss) (Inno Setup, опционально).

## Как собрать EXE

Команды из папки `source/`:

1. Обновите версию в файле `VERSION`.
2. Установите зависимости:

```powershell
pip install -r requirements.txt
```

3. Соберите exe:

```powershell
pyinstaller --noconfirm USBBackupApp.spec
```

Готовый файл: `dist\USBBackupApp.exe`.

## Как собрать ZIP для клиента

Из **корня репозитория** (на уровень выше `source/`):

```powershell
python build_sale_package.py --version 1.0
```

Параметры по желанию:

- `--root` — корень проекта, если скрипт запускается из другого каталога
- `--exe` — явный путь к `USBBackupApp.exe`
- `--out-dir` — каталог для итогового ZIP (по умолчанию корень репозитория)
- `--staging-dir` — база для временной папки `sale_staging` (по умолчанию `sale_staging` в корне)

Перед сборкой убедитесь, что в корне есть актуальные `screenshots/*.png` и файлы `README_RU.md`, `README_CLIENT.md`, `SUPPORT.md`, `config/app_state.json.example`.

## Установщик Inno Setup (опционально)

1. Установите [Inno Setup](https://jrsoftware.org/isinfo.php).
2. Из корня репозитория выполните:

```text
ISCC.exe USBBackupApp.iss
```

Результат: `installer_output\USBBackupApp_Setup_1.0.exe` (номер версии совпадает с `#define MyAppVersion` в `.iss`).

## Примечания

- Windows SmartScreen может предупреждать о запуске неподписанного exe.
- Для снижения предупреждений используют **код-подпись** (Code Signing Certificate).
- В архив для клиента **не включайте** личные `app_state.json`, логи и папки `build`/`dist` целиком — только результат `build_sale_package.py`.
- Расширенное руководство разработчика: [`docs/USER_GUIDE.md`](../docs/USER_GUIDE.md), коммерческая EULA при необходимости: [`docs/EULA.txt`](../docs/EULA.txt).

## FAQ

**На каких версиях Windows работает программа?**  
Windows 10 и Windows 11.

**Нужен ли интернет для работы?**  
Нет.

**Что делает программа?**  
Сравнивает флешку и папку бэкапа и предлагает действие: полный бэкап, обновление папки бэкапа или загрузка отличий на флешку. Прогресс — в нижней панели главного окна.

**Можно ли выбрать любую папку для бэкапа?**  
Да.

**Что будет, если флешку переименовать?**  
Автозапуск ориентируется на серийный номер тома, переименование метки не ломает привязку.

**Нужны ли права администратора?**  
Для обычного копирования — не обязательно. Для настройки автозапуска — нужны.

**Программа удаляет файлы?**  
Основная логика — копирование и синхронизация. Тестируйте на копии данных.

**Что делать, если автозапуск не сработал сразу?**  
Запустите приложение от имени администратора, заново включите автозапуск, переподключите флешку. Автозапуск использует задачу Планировщика **`USBBackupApp_WMI_Daemon`** (логон) и WMI‑демон для обнаружения вставки флешки; см. FAQ в [`README_RU.md`](../README_RU.md).

**Где логи?**  
Журнал в правой панели окна; отдельные log-файлы в релизе не создаются.
