# USB Backup App — руководство пользователя

Версия: см. заголовок окна приложения.

## Быстрый старт

1. Подключите флешку или диск, который нужно бэкапить.
2. В поле **Источник (USB/диск)** выберите устройство из списка (кнопка **Обновить** — если устройство не появилось).
3. В поле **Папка для бэкапа** укажите папку на компьютере (кнопка **Обзор**).
4. Нажмите **Сканировать папку**.
5. После анализа в нижней панели появится статус. Приложение само предложит подходящий режим:
   - **СДЕЛАТЬ ПОЛНЫЙ БЭКАП** — если папка бэкапа пуста или сильно отличается от флешки.
   - **ОБНОВИТЬ ПАПКУ БЭКАПА** — скопировать новые/изменённые файлы с флешки в папку бэкапа.
   - **ЗАГРУЗИТЬ ОТЛИЧИЯ НА ФЛЕШКУ** — перенести изменения из папки бэкапа обратно на флешку.
6. Нажмите нужную кнопку и дождитесь завершения операции.

Весь прогресс (процент, текущий файл, скорость, оставшееся время) отображается в нижней панели главного окна. Для отмены нажмите кнопку **Остановить** в той же панели.

## Права администратора (важно)

Некоторые функции (например, настройка автозапуска) требуют прав администратора.
Если приложение запущено без прав администратора, Windows может показать запрос UAC — подтвердите его.

## Автозапуск по флешке

Блок **Автозапуск по флешке** позволяет включить автоматический запуск сканирования при подключении выбранной флешки. Для настройки требуются права администратора.

После включения автозапуска отключите и снова подключите флешку — приложение откроется и автоматически запустит сканирование.

**Как это устроено:** при входе в Windows **Планировщик заданий** запускает фоновый режим приложения (WMI‑демон). Уже запущенный демон через **WMI** отслеживает подключение томов и открывает окно программы только для флешки с нужным серийным номером. Задача в Планировщике называется **`USBBackupApp_WMI_Daemon`**. Подробные шаги «включить / отключить» и ручное удаление задачи — в клиентском [`README_RU.md`](../README_RU.md) (раздел FAQ).

## Частые проблемы

### Приложение «не запускается» из исходников

Если вы запускаете `python src\main.py` без прав администратора, приложение перезапустит себя через UAC. Подтвердите запрос.
Если UAC не появляется, запустите PowerShell **от имени администратора** и попробуйте снова.

### Сканирование «долго висит»

После этапа сканирования начинается этап **анализа отличий** — это нормально. В нижней панели главного окна отображается статус «Анализ…» и текущий файл.

### Файлы пропущены при копировании

Если некоторые файлы были пропущены (например, заняты другим процессом или находятся в синхронизируемой папке OneDrive), по завершении операции появится предупреждение со списком пропущенных файлов.
