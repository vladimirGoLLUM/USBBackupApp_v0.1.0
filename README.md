# USB Backup App

Простое Windows-приложение для резервного копирования данных между флешкой и папкой бэкапа.

## Что умеет

- Автоматически находит съёмные USB-устройства и внешние диски.
- Сравнивает флешку и папку бэкапа, определяет режим операции.
- Включает нужные действия по ситуации:
  - `СДЕЛАТЬ ПОЛНЫЙ БЭКАП` — если папка бэкапа пуста или не похожа на флешку.
  - `ОБНОВИТЬ ПАПКУ БЭКАПА` — копирует новые/изменённые файлы с флешки в папку бэкапа.
  - `ЗАГРУЗИТЬ ОТЛИЧИЯ НА ФЛЕШКУ` — копирует отличия из папки бэкапа на флешку.
- Показывает прогресс, скорость и оставшееся время в нижней панели главного окна.
- Поддерживает отмену операции в любой момент.
- Ведёт журнал операций в интерфейсе.
- Опциональный автозапуск при подключении конкретной флешки (через Windows Task Scheduler).

## 📥 Как скачать и запустить (Готовая версия)

Если вы не планируете изменять код, используйте готовую сборку для Windows:

1. Перейдите в раздел [**Releases**](https://github.com/pycraft-dev/USBBackupApp_v0.1.0/releases/latest).
2. Скачайте архив `USBBackupApp_v0.1.0.zip`.
3. Распакуйте архив и запустите файл `USBBackupApp.exe`.

*Примечание: При первом запуске Windows SmartScreen может выдать предупреждение (так как файл не имеет цифровой подписи). Нажмите "Подробнее" -> "Выполнить в любом случае".*

## Как пользоваться

1. Запустите программу.
2. Выберите флешку в поле `Источник (USB/диск)`.
3. Укажите папку в поле `Папка для бэкапа` (кнопка `Обзор`).
4. Нажмите `Сканировать папку`.
5. Нажмите доступную кнопку нужного действия.
6. Дождитесь окончания — прогресс виден в нижней панели главного окна.

## Запуск из исходников

Требования:

- Windows 10 / 11
- Python 3.11+

Установка зависимостей:

```powershell
pip install -r requirements.txt
```

Команда запуска (из папки `source/`):

```powershell
python src\main.py
```

## Сборка EXE

Из папки `source/`:

```powershell
pyinstaller --noconfirm USBBackupApp.spec
```

Готовый файл: `dist\USBBackupApp.exe`.

## Важно

- Не отключайте флешку во время копирования.
- Для корректной работы используйте локальные диски/папки с полным доступом.
- Если папка бэкапа находится в синхронизируемом каталоге (OneDrive и др.), возможны редкие ошибки доступа к отдельным файлам — они пропускаются с предупреждением.
