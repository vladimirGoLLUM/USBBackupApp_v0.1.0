# USB Backup App

Современное Windows‑приложение для резервного копирования данных между **USB‑накопителем** и **папкой бэкапа**.

## Установка и запуск

### Готовая сборка (EXE)

1. Откройте [Releases](https://github.com/pycraft-dev/USBBackupApp_v0.1.0/releases/latest).
2. Скачайте архив `USBBackupApp_v0.1.0.zip`.
3. Распакуйте архив и запустите `USBBackupApp.exe`.

> При первом запуске Windows SmartScreen может показать предупреждение (файл без цифровой подписи).

### Запуск из исходников

Требования:

- Windows 10 / 11
- Python 3.11+

Установка зависимостей (из папки `source/`):

```powershell
pip install -r requirements.txt
```

Запуск (из папки `source/`):

```powershell
python src\main.py
```

### Сборка EXE

Из папки `source/`:

```powershell
pyinstaller --noconfirm USBBackupApp.spec
```

Готовый файл: `dist\USBBackupApp.exe`.

## Язык интерфейса

В правом верхнем углу главного окна можно выбрать **RU** или **EN**. Выбор сохраняется в файле состояния `app_state.json` (поле `language`: `ru` или `en`) рядом с путём последней папки бэкапа. В режиме разработки файл лежит в [`source/config/app_state.json`](source/config/app_state.json); в установленной сборке — в каталоге профиля пользователя (см. `resolve_app_state_path` в коде). Словари переводов — [`source/data/translations.json`](source/data/translations.json).

## Документация

### Быстрый старт

1. Запустите программу.
2. Выберите флешку в поле **Источник (USB/диск)**.
3. Укажите папку в поле **Папка для бэкапа** (кнопка **Обзор**).
4. Нажмите **Сканировать папку**.
5. Нажмите доступную кнопку действия:
   - **СДЕЛАТЬ ПОЛНЫЙ БЭКАП**
   - **ОБНОВИТЬ ПАПКУ БЭКАПА**
   - **ЗАГРУЗИТЬ ОТЛИЧИЯ НА ФЛЕШКУ**

### Автозапуск для конкретной флешки (Windows Task Scheduler + WMI)

Автозапуск реализован через задачу Планировщика **при входе в Windows**, которая запускает лёгкий WMI‑демон.
Демон слушает события вставки томов и открывает UI **только если вставлена целевая флешка** (по серийнику тома).

- Включение/выключение автозапуска делается из интерфейса (нужны права администратора).
- Работает корректно даже если одновременно подключены другие флешки.

### Безопасность и надёжность

- Приложение не использует `print()` в продакшен‑коде.
- Все операции копирования выполняются с отображением прогресса в UI и возможностью отмены.

│   ├── device_detector.py     # Определение устройств (Volume Serial HEX)
